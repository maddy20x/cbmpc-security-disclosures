# Contributing to CB-MPC Security Disclosures

## How to Contribute

This repository is for responsible security research disclosure. We welcome contributions that help improve the security of the CB-MPC library.

### Reporting New Vulnerabilities

1. **Do NOT disclose vulnerabilities publicly** until they are fixed
2. Submit findings through the Coinbase Bug Bounty Program
3. Use this repository only for validated, disclosed vulnerabilities

### Improving Documentation

1. Fork the repository
2. Create a feature branch: `git checkout -b improve-docs`
3. Make your changes
4. Submit a pull request

### Adding Proof of Concept

1. Ensure your PoC is reproducible
2. Include compilation instructions
3. Document expected output

## Code of Conduct

- Be respectful and professional
- Do not exploit vulnerabilities in production systems
- Follow responsible disclosure practices

## License

This repository is for educational and security research purposes only.
